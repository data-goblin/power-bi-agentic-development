# v26.30.0

## Faster notebook deploys: poll the definition LRO tightly

`fabric-cli` now ships `scripts/deploy_notebook.py` for creating or updating a notebook's definition, and the skills call out the reason it matters: the poll interval on the create/update long-running operation is the single biggest performance lever for definition changes.

- `fab import`, `nb create`, and `nb cell edit` poll the LRO at the server's advertised `Retry-After: 20`, so a definition change takes 25-60s even though the server-side work finishes in ~1s. Neither CLI exposes a poll-interval knob.
- `deploy_notebook.py` polls at ~0.3s (`--poll-interval`), auto-detects create vs update, and brings the same deploy down to ~1-2s.
- The `fabric-cli` skill, `references/import-download-deploy.md`, and `references/notebooks.md` document the gotcha, a benchmark, and usage.
- The `etl` `executing-spark` skill points to the script for persisting code as a notebook.

## Reading a notebook's real verdict

A notebook run via `fab job run` reports `Completed` even when the notebook caught an exception and exited a failure payload.

- New `fabric-cli` `scripts/run_notebook_checked.py` reads the notebook's exit value to surface the true pass/fail outcome.
- The skills document the SQL-endpoint `refreshMetadata` sync needed after a Spark write so a freshly written table is queryable.

## Removed: pbiad CLI

The `pbiad` Rust CLI source has been removed from this marketplace repo and now lives in its own repository. `/tools/*` stays gitignored and local-only; tools are not committed back into this repo.
