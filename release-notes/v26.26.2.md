# v26.26.2

Adds the **goblin-mode** plugin, consolidates the old **agentic-help** plugin into it, and folds the old `/audit-context` command into it. goblin-mode is now the one place to get started, set up your tools, and audit and improve your whole agentic setup.

**Consolidation**

- The `agentic-help` plugin is removed and its `setup-agentic-dev` install workflow is folded into the new `help-me-get-started` skill in `goblin-mode`, so getting started is one skill instead of two.
- New skill `help-me-get-started`: a slow, friendly, jargon-free guide to agentic development for someone new to agents, especially a Power BI or data person. It talks through what they want to do and why, feels out their role and access (report author vs modeler vs admin, licensing tier, API and machine access, other systems), teaches the five pillars (model, context, prompt, tools, environment) one small step at a time with local interactive HTML explainers opened in the browser, and checks and installs what they need. It adapts pace: a full tutorial for a nervous beginner, or a fast install run for someone who just wants the commands (the folded `install.md` plus per-tool references cover Windows and macOS). Bundles `show_explainer.py` for the visual explainers. Opinionated core: start tiny, own your context, don't fear the terminal.

**New skill: improve-my-agent-setup** (in goblin-mode)

- Skill `improve-my-agent-setup` (also triggers on "audit my setup", "goblin mode", and similar): a setup-wide health check that measures what it can, interviews for what it can't, writes a scored report to `.claude/scratchpads/`, and then offers to fix the weak spots one at a time.
- Four modes via argument (`/improve-my-agent-setup deep`): `shallow` (quick glance), `deep` (routine checkup, the default), `ultra` (machine-wide, blunt, high effort), `yolo` (mines transcripts, drill-sergeant, demolition-and-rebuild). The mode sets depth, interview cadence, and tone; see `references/modes.md`.
- Deterministic `scan_setup.py` (stdlib-only, metadata-only) gathers facts across skills, memory and its organization, harness config, tools, speech-to-text input, local/OSS harnesses and model runners, other clients (Claude Desktop, VS Code, Cursor, Zed), git, transcripts, execution environment and isolation, permission mode, supply-chain guards, declared-vs-enforced safety rules, and a secrets/PII sweep. It never writes secret material to disk; secrets are reported by location and kind only.
- Scan breadth by scope: `user` (default), `project` (one repo), `deep` (wide sweep across `~/.config`, `~/.aws`, `~/.azure`, `~/.ssh`; warned as slow and token-heavy). Modes map onto these.
- Twenty themed rubrics in `references/dimensions.md`, each written as good / bad / nuance / actions, covering skills and ownership, memory and context, tools and redundancy, input ergonomics (STT), model independence, cross-client redundancy, harness config, version control, permission posture, execution environment and the isolation ladder, safety enforcement, supply-chain defense, secrets hygiene, delegation ethics, determinism, focus and shipping, model and effort, usage review, automation, and knowledge systems.
- Strongly advises scheduling a recurring re-audit with `/schedule` (daily / weekly / biweekly) so drift is caught while it's cheap.
- `references/context-review.md` carries the full context-file audit that `/audit-context` used to do.
- `references/fix-catalog.md` gives the concrete fixes with commands and reasoning, including the op-service-account + read-blocking-hook pattern for secrets, the isolation ladder and trust-domain separation, backing naked safety rules with real enforcement, STT adoption, and a local/OSS model-independence hedge.

**Removed**

- The `/audit-context` command in `fabric-cli` is gone; its behavior now lives in `goblin-mode`.

**Versioning**

- Lockstep bump of all plugins and the marketplace to 26.26.2.
