# v26.29

Small release, one focus: the `te-cli` skill now matches CLI 0.6.0, with every
command claim verified against the actual binary instead of the docs site.

- Corrected the things the binary disproved: `te diff` exit codes, `te query`
  taking `--file` (not `-f`) and needing a deployed model, `te vertipaq`
  needing a workspace or a `.vpax` import, the real `te macro set` properties,
  and the real `te test snapshot`/`compare` flags.
- `te move` cascade behavior is now documented from tests, not folklore:
  renames rewrite bracketed DAX references for measures and tables;
  cross-table moves do not, and the save gate rejects them.
- New `references/testing.md`: how to author `.test.yaml` suites (all
  assertion types, tolerance, tags, matrix expansion), snapshot regression,
  and A/B comparison between two deployed models.
- Rebuilt CI/CD guidance around two pipelines and one artifact: PR validation
  with no secrets, then deploy plus post-deploy regression tests per
  environment. Includes Fabric service-principal and XMLA setup, a public-CDN
  runner install step, and full GitHub Actions and Azure DevOps examples.
- `get-te-cli.md` now warns that the CDN rejects HEAD requests; probe with a
  ranged GET.

Everything stays lockstep on `26.29.0`.
