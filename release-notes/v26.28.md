# v26.28

Adds the **pbiad** CLI to the repo, ships AI-metadata tooling for semantic models, sharpens every plugin's marketplace description, and drops the unrecognized `version:` field from skill frontmatter.

**New tool: pbiad**

- The `tools/pbiad/` Rust CLI (Power BI Agentic Development) is now tracked in the repo. It installs, moves, audits, and manages this marketplace's skills across supported agents.
- `skills manage` is now scope-aware: the interactive tree groups each skill under its target scope, so user-scoped skills no longer appear in project sections and vice versa. Moving a skill hops it cleanly to the destination section.
- Manage labels no longer repeat the scope word already shown by the User/Project section headers; they read as `(keep)`, `(move)`, `(install)`, `(installed)`, `(deactivate)`, or `(uninstall)`.

**AI metadata tooling**

- New `get_semantic_model_ai_metadata.py` (in `semantic-models/semantic-model` and `fabric-cli/fabric-cli`) reads AI instructions and Copilot data schema from Fabric semantic-model definitions.
- New `te-cli` scripts for AI metadata: `manage-ai-metadata.csx` (non-interactive CustomInstructions and Copilot schema Entities), plus `edit-ai-instructions-interactive.csx` and `edit-ai-schema-interactive.csx` for TE3 Desktop.
- The `semantic-model`, `fabric-cli`, and `te-cli` skills reference these scripts.

**Descriptions**

- Rewrote marketplace and plugin descriptions across every plugin for clarity and consistency.

**Cleanup**

- Removed the unrecognized `version:` field from SKILL.md frontmatter across all skills.

**Versioning**

- Lockstep bump of all plugins and the marketplace to 26.28.0.
