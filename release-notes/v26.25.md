# v26.25

The chunky one: three new plugins and a `pbir` CLI that grew a few new tricks.

**New plugins**

- **paginated-reports**: RDL from the terminal. It interviews you about what you actually need, writes the report, renders it to check its own work, and validates every `.rdl` you touch. Yes, paginated reports. We went there.
- **custom-visuals**: Deneb, Python, R, SVG, and the whole pbiviz toolchain (with an MCP) move out of `reports` into their own room. Only worth installing if you actually build custom visuals.
- **etl**: run Spark on Fabric over the Livy API with no notebook, or point DuckDB straight at a lakehouse. Mostly redundant with `fabric-cli`, which already does this, so reach for it only if you want the ETL-focused framing.

**The pbir CLI glow-up**

- New verbs: `color` and `fonts` (audit and swap a value across the report and its theme in one pass), `usage` (who actually opens this report), and desktop reload + screenshot so you can eyeball edits on the live canvas.
- `pbir validate --semantic` checks visual type ids and object names against the core visual catalog, so a typo gets caught instead of quietly refusing to render. A new `pbir-format` validation reference gathers every conformance check in one place.

**Also worth a nod**

- `pbi-report-design` is now the shared design canon, with a locked design brief and a re-theming workflow so a report commits to one look and keeps it.
