# v0.12.0

- Add pre-flight check for `pbi-search` CLI in te-docs skill, and alternative without CLI. [`pbi-search`](https://github.com/data-goblin/pbi-search) is a separate experimental project testing different CLI-based search methods, currently using lexical scoring but also testing semantic search and vector-based approaches to find ways to enable agents to be better at getting relevant context on-the-fly
- New hooks in pbip plugin that catch invalid PBIR and TMDL edits made via Bash commands (python3, sed, jq, awk, etc.), complementing the existing Write/Edit hooks
- Uses the new `if` condition feature to only fire on relevant Bash commands

**Known limitation:** Bash redirect targets are not caught. For example:

```bash
# Caught (file path is a direct argument)
python3 script.py tmp/MyReport.Report/definition/report.json
sed -i '' 's/x/y/' tmp/MyReport.Report/definition/report.json

# NOT caught (file path follows a redirect operator)
echo '{}' > tmp/MyReport.Report/definition/report.json
jq '.x = 1' input.json > tmp/MyReport.Report/definition/visual.json
```
