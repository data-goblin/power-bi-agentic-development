# v26.26

The re-organization line. Skills get consolidated and lean on explicit invocation more than autopilot; if you want the old always-on behavior, pin to 26.25. More lands here as the reorg rolls out.

**New plugin**

- **agentic-help**: the on-ramp. It installs and sets up the tools the other plugins expect (the CLIs, MCP servers, auth) so a fresh machine gets to a working agentic Power BI and Fabric setup without a scavenger hunt. Start here if nothing else is wired up yet.

**fabric-cli: a hosted SQL route**

- New `fabric-sql` MCP server option for T-SQL against a Lakehouse SQL endpoint, Warehouse, or SQL Database. It runs server-side (no local `sqlcmd`, DuckDB, or Spark) and hands back CSV, so it is the convenient route when you just want to fire a query.
- The `querying-data` reference now ranks the four T-SQL paths by tradeoff: `fabric-sql` MCP for convenience, `sqlcmd` for speed and reliability, DuckDB to read Delta straight off OneLake, PySpark for writes and distributed work. The MCP is in preview and adds a gateway hop, so `sqlcmd` stays the default when latency or reliability matters.
- Auth is a bearer header on the Power BI scope; Claude Code's interactive `/mcp` OAuth does not work here because Entra refuses dynamic client registration. Export a token and reference it from the server headers, or register a dedicated public-client app for self-refresh.
- New `query_sql_mcp.py` script hits the same hosted endpoint directly over JSON-RPC (no MCP client, no interactive OAuth); it mints a token from `az login`, kept in memory.
